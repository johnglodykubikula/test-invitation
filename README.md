## Reflection

### 1. Interpreting User Needs
I interpret user needs by breaking down requirements into smaller tasks and focusing on what the user is trying to achieve. Creating user stories helps me understand the user’s perspective and prioritize features based on value.

### 2. Approach to Developing Programs
When developing programs, I like to follow an Agile approach by working in small iterations and continuously improving the product. I plan to keep using practices like sprint planning, daily communication, and retrospectives to improve my workflow and adapt to changes.

### 3. Being a Good Team Member
Being a good team member means communicating clearly, being reliable, and supporting others when needed. It also means staying open to feedback and making sure the team is aligned toward the same goal.
